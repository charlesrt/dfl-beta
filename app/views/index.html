Design prototypes for

Search enhancements

Different prototypes have been created to test specific scenarios and interactions for how we might implement solutions that work in the future.

Before you start:

  • This is not a real product or service. You should only continue if you have been invited to test this prototype.
  • This prototype is intended to communicate and test functional ideas only. All visual design is illustrative only for prototyping purposes and not to be used for comparison to delivered product(s).
  • Data entered is stored locally only for the duration of your session. If you need to you can clear data.

Prototypes

Users need to

Understand the logic behind search

Highlight search strings within results to increase result relevancy Ready

Research discussion:

  • Do users understand the relationship between what was searched and the results?
  • What do users assume of search results without highlighting?

Increase visibility of selected filters within search results Ready

Research discussion:

  • Do users understand the faceted view of the results?
  • Do users understand the AND/OR relationship between facets?
  • How do users react to knowing that some relevant results aren't presented under a filtered view because those filters aren't available in the metadata?
  • Do users know how to remove filters?

Advanced search logic for specificity including operators such as AND and OR Ready

Research discussion:

  • Are user familiar with the formatting of advanced search queries using operators?
  • If not, do users understand how they would use them based on the content available?
  • Do users use the quick links or follow the examples given?
